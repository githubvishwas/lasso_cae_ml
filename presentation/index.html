What is the criterion for reduction?

Reduce dimensions but keep the most important information

What does that mean?

Information Preservation

Distances in high-dimensional space need to be kept in Low-dimensional space (according to some information metric).

Largest distances are preserved well
→ most important information is kept

The metric defines what is important and what not!!!!!

Topic #2: Embedding Difficulties

Incompatible Meshes

Difficulty Levels
  1. Rougher, finer and remeshed Meshes
  2. Different geometric sizes (holes, width, ...)
  3. Different topology (hole/no hole, additional rib, ...)
  4. Strong morphing of components
  5. Different parts in a substructure (replace parts)
  6. Different substructures (compare car A vs. car B)

Solution Approaches

1-3: Nearest Neighbor Mapping
4-5: Advanced Mapping or Projection Schema

Different Mesh Resolutions

Advice: Rougher mesh as baseline

Different Geometric Sizes

Advice: Take geometry with best coverage (smallest holes) as baseline (100% model)

Topology Differences

Take as reference a 150% model

Advanced Mapping:

Map region to region

Solutions for Different Meshes

  1. Nearest Neighbor Mapping
  2. Advanced Mapping Techniques

Nearest Neighbor Mapping

Easiest:

Transfer value to closest point
Roughly same element size

Medium:

Map according to weighted distance
Large element size differences

Hardest:

Map according to Ansatzfunction
Large element size differences

Advanced Mapping Techniques

Mapping through for example spectral descriptors can align two deformed geometries according to feature points

Deep Spectral Descriptors: Learning the point-wise correspondence metric via Siamese deep neural networks, Z. Sun et al., 2018

Topic #3: Projection Methods

Project results from meshes onto something comparable

Line Projection Schema

  1. Put a line through a component
  2. Project results onto line
  3. Smooth/Locally average results along line

Result Hashing

Distribution Function of Plastic Strain

Hash Comparison

Result hashes (here functions) can be compared in many ways. In this case the similarity between functions is evaluated by using a (function) dot product.

Sphere Projection Schema

  1. Project results onto a surrounding sphere
  2. Average values in zones which yields a matrix
  3. Compare hash matrices
